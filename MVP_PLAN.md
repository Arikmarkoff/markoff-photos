## Что такое MVP для этого проекта

MVP (минимально жизнеспособный продукт) — это версия сайта, которую не стыдно отправить первому клиенту. Не идеальная, но рабочая: открывается, выглядит хорошо, есть способ связаться.

Для Markoff Photos MVP = 3 страницы + рабочие ссылки + реальный контакт.

## Шаги MVP (от простого к сложному)

---

### ШАГ 1 — Страница портфолио `portfolio.html`
**Что делаем:** Создаём отдельную страницу с расширенным портфолио, разбитым по 5 категориям.

**Почему первым:** Это самая частая причина, по которой клиент открывает второй экран — он хочет посмотреть больше работ.

**Структура страницы:**
- Шапка (та же, что в index.html — единый стиль)
- Hero — маленький, с заголовком «Портфолио» и подписью
- Фильтр-табы: Все / Портрет / Семья / Лав стори / Коммерция / Предметка
- Галерея — сетка 3 колонки, при клике → lightbox
- CTA внизу — «Понравились работы? Обсудим вашу съёмку»
- Футер (тот же)

**Видимый результат:** Открываем в браузере — видим страницу с галереей и фильтрами.

**Проверка:** Клик по фото → открывается lightbox. Клик по табу → фильтрует карточки.

---

### ШАГ 2 — Страница контактов `contact.html`
**Что делаем:** Страница с формой обратной связи, контактами и картой.

**Почему вторым:** Клиент, готовый написать, должен попасть на специальную страницу — это повышает конверсию vs просто кнопка Telegram.

**Структура:**
- Шапка
- Заголовок: «Давайте знакомиться» + подзаголовок
- 2 колонки: форма слева, контакты справа
- Форма: имя + тип съёмки (select) + желаемая дата + Telegram/телефон + кнопка
- Контакты: Telegram, Instagram, ВКонтакте, телефон, время ответа
- После отправки формы — toast «Отлично! Напишу вам в течение часа»
- Футер

**Видимый результат:** Форма работает, при отправке — уведомление.

**Проверка:** Заполнить форму → нажать → появился toast.

---

### ШАГ 3 — Связать все страницы навигацией
**Что делаем:** Обновить шапку и футер во всех трёх файлах — добавить ссылки между страницами.

**Почему третьим:** До этого шага каждая страница — остров. После — это уже сайт.

**Что меняем:**
- В шапке `index.html`: добавить «Портфолио» → `portfolio.html`, «Контакты» → `contact.html`
- В `portfolio.html` и `contact.html`: шапка ведёт обратно на главную
- В футере всех страниц — единые ссылки

**Видимый результат:** Можно ходить между страницами как на настоящем сайте.

**Проверка:** Открыть index.html → клик «Портфолио» → открылась portfolio.html → клик «Главная» → вернулись.

---

### ШАГ 4 — Наполнить реальным контентом
**Что делаем:** Заменяем заглушки на настоящие данные.

**Список замен (подсказка для себя):**
```
t.me/username          → реальный Telegram
+7 (999) 123-45-67    → реальный телефон
instagram.com          → реальная ссылка
vk.com                 → реальная ссылка
«7 лет в фотографии»   → реальная цифра
«400+ съёмок»          → реальная цифра
цены «от 8000»         → реальные цены
текст «О себе»         → реальный текст
```

**Видимый результат:** Кнопка Telegram ведёт к реальному чату.

**Проверка:** Нажать кнопку «Написать в Telegram» → открылся реальный диалог.

---

### ШАГ 5 — Добавить реальные фото
**Что делаем:** Заменяем цветные градиенты-заглушки на настоящие фотографии.

**Технические требования:**
- Формат: WebP (конвертировать через squoosh.app)
- Вес: ≤ 400 КБ для hero, ≤ 200 КБ для галереи
- Размер: 2400px по длинной стороне для hero, 1200px для карточек
- Структура папок:
  ```
  images/
  ├── hero/          — фото для первого экрана
  ├── portfolio/     — работы для галереи
  │   ├── portrait/
  │   ├── family/
  │   ├── lovestory/
  │   ├── commercial/
  │   └── product/
  └── about/         — фото фотографа
  ```

**Видимый результат:** Вместо серых градиентов — реальные фотографии.

**Проверка:** Открыть на телефоне — фото грузятся быстро (< 3 сек).

---

### ШАГ 6 — Финальная проверка перед запуском
**Что делаем:** Тестируем всё перед публикацией.

**Чек-лист:**
- [ ] Все ссылки кликабельны и ведут куда надо
- [ ] Telegram-кнопки открывают реальный чат
- [ ] Телефон набирается при клике
- [ ] Сайт открывается на iPhone (Safari) — ничего не сломано
- [ ] Сайт открывается на Android (Chrome) — ничего не сломано
- [ ] Форма отправляет и показывает уведомление
- [ ] Фото не размытые и не пиксельные
- [ ] PageSpeed Insights (pagespeed.web.dev) — Score > 70

## Принципы работы (напоминание себе)

1. Каждый файл — самодостаточный HTML (CSS и JS внутри)
2. Без внешних библиотек — только HTML + CSS + JS
3. Сначала объяснить что делаю простым языком — потом писать код
4. После каждого шага — проверить в браузере
5. Светлая тема, оранжевый акцент `#E8652A`, шрифты Cormorant + Inter
