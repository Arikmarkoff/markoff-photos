# Правила работы с проектом Markoff Photos

## Архитектура

- Каждая страница — один самодостаточный HTML-файл (CSS и JS внутри, в тегах `<style>` и `<script>`)
- Никаких внешних библиотек — только чистый HTML + CSS + JavaScript
- Шрифты подключаются через Google Fonts (это допустимо, не библиотека)
- Файлы называются по-английски: `index.html`, `portfolio.html`, `about.html` и т.д.

## Дизайн

**Цветовая схема — светлая (текущая в index.html):**

| Роль | HEX |
|---|---|
| Основной фон | `#FAF8F5` (молочный белый) |
| Вторичный фон | `#F0EDE8` (тёплый светло-серый) |
| Основной текст | `#2C1A0E` (тёмно-коричневый) |
| Вторичный текст | `#8C7B72` (тёплый серый) |
| Акцентный цвет | `#E8652A` (оранжевый) |
| Акцент hover | `#FF7A3D` |
| Тёмные секции (footer, CTA) | `#1A110A` |
| Разделители | `rgba(44,26,14,0.1)` |

**Типографика:**
- Заголовки: `Cormorant Garamond`, тонкий вес (300–400), крупно
- Текст: `Inter`, вес 400, размер 15–16px, line-height 1.7
- Метки / теги: `Inter`, uppercase, letter-spacing 0.12em, 11px

**Компоненты:**
- Скругления: 8–12px на карточках
- Кнопки: border-radius 4px, padding 14px 32px, uppercase, letter-spacing 0.1em
- Основная кнопка: оранжевый фон `#E8652A`, белый текст → при hover: transparent + оранжевая рамка
- Outline кнопка: transparent фон, белая рамка → при hover: белый фон, тёмный текст

## Адаптивная вёрстка

- Mobile-first: сначала телефон, потом расширяем на десктоп
- Три брейкпоинта:
  - Мобайл: до 768px
  - Планшет: 768px – 1024px
  - Десктоп: от 1024px
- На мобайле:
  - Все grid → одна колонка (или максимум две)
  - Шрифты уменьшаются через `clamp()`
  - Боковые отступы контейнера: 20px
  - Sticky-кнопка Telegram внизу экрана
  - Кастомный курсор отключён (`cursor: auto`)
- Минимальный размер шрифта: 14px
- Touch-таргеты (кнопки, ссылки): минимум 44×44px

## Стиль текста

- **Простой и дружеский** — пишем как живой человек, не как коммерческий буклет
- Без канцелярита: не «осуществляю профессиональный подход», а «работаю так, чтобы вы забыли про камеру»
- Без пафоса: не «уникальный», не «высочайшее качество», не «индивидуальный подход»
- Короткие предложения — лучше два коротких, чем одно длинное
- CTA-кнопки — с глаголом: «Написать», «Узнать стоимость», «Обсудить съёмку»
- Обращение: на «вы» (с маленькой буквы)

## Порядок работы

**Перед созданием нового файла или крупным изменением — сначала описать план:**
- Что именно будет создано / изменено
- Из каких секций состоит (если это страница)
- Какие интерактивные элементы будут добавлены
- Дождаться подтверждения, затем делать

**Перед редактированием существующего файла:**
- Прочитать файл целиком (или нужную часть)
- Не удалять существующий рабочий код без явной просьбы
- Изменения делать точечно через Edit, не переписывать весь файл

## Что не делаем

- Не подключаем jQuery, Bootstrap, GSAP, Swiper и другие библиотеки
- Не создаём отдельные CSS/JS файлы — всё внутри HTML
- Не добавляем функции «на будущее» — только то, что нужно сейчас
- Не пишем длинные комментарии к очевидному коду
- Не меняем светлую тему на тёмную — проект на светлой основе (тёмные только footer и финальный CTA-блок)
