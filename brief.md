# ТЕХНИЧЕСКОЕ ЗАДАНИЕ
## Сайт-портфолио фотографа | Markoff Photos

> **Версия:** 1.0  
> **Дата:** апрель 2026  
> **Основан на:** research.md (анализ 5 конкурентов, тренды 2025–2026, UX-психология)

---

## 1. О ПРОЕКТЕ

### Проект
Одностраничный лендинг-портфолио фотографа. Главная цель — конвертировать посетителя в клиента через Telegram.

### Специализации
- Портретная съёмка
- Семейная съёмка
- Лав стори
- Коммерческая съёмка
- Предметная съёмка

### Цель сайта
Привлекать частных клиентов и студии, показывать стиль и уровень работ, формировать доверие, снимать возражения и направлять к единственному действию — написать в Telegram.

---

## 2. ЦЕЛЕВАЯ АУДИТОРИЯ

### Сегмент A — частные клиенты
- Пары, планирующие лав стори или свадьбу
- Семьи с детьми (от 0 до школьного возраста)
- Люди, ищущие портрет для личного бренда / соцсетей
- Возраст: 25–45 лет, преимущественно женщины — инициаторы съёмки
- Канал поиска: Instagram → переход на сайт; Яндекс/Google «фотограф [город]»

### Сегмент B — бизнес-клиенты
- Малый бизнес для предметной и коммерческой съёмки
- Бренды, магазины, кафе, цветочные студии
- Принимают решение быстро, смотрят на портфолио и цены

### Боли и страхи клиента
- «Я не умею фотографироваться, буду выглядеть деревянно»
- «Вдруг дорого, а я не знаю бюджет»
- «Не знаю, как проходит съёмка и что делать»
- «А вдруг мне не понравятся результаты?»

### Психологические триггеры (из research.md)
1. Идентификация — клиент видит себя в тексте и фото
2. Социальное доказательство — реальные отзывы с историями
3. Авторитет — конкретные цифры (лет опыта, количество съёмок)
4. Дефицит — ограниченное количество мест в месяц
5. Снижение риска — прозрачный процесс, бесплатная консультация

## 4. СЕКЦИИ — ПОДРОБНОЕ ТЗ

---

### [00] ШАПКА (HEADER)
**Тип:** фиксированная, при скролле — blur-эффект (backdrop-filter: blur)

**Элементы:**
- Слева: имя/логотип (текстовый) — `MARKOFF` или `Аркадий Марков`
- Справа: навигация — `Портфолио` / `Обо мне` / `Стоимость` / `Контакт`
- Крайний правый элемент: кнопка `Написать в Telegram` — основной CTA, всегда видна

**Мобильная версия:** бургер-меню, кнопка Telegram — фиксированная внизу экрана (sticky)

**Поведение при скролле:**
- Фон шапки становится полупрозрачным с blur
- Логотип и навигация остаются читаемы на любом фоне

---

### [01] HERO — ПЕРВЫЙ ЭКРАН
**Цель:** захватить внимание за 3 секунды, передать стиль, обозначить «для кого»

**Макет:**
- Полноэкранное фоновое фото (100vw × 100vh) — лучшая работа, передающая настроение
- Лёгкое тёмное затемнение поверх фото (overlay: rgba(0,0,0, 0.25))
- Контент выровнен по нижнему левому углу или по центру (обсудить)

**Тексты:**

| Элемент | Текст (вариант — заменить на финальный) |
|---|---|
| H1 — главный заголовок | `Снимаю то, что нельзя повторить` |
| Подзаголовок | `Портрет · Семья · Лав стори · Коммерция · Предметка` |
| Вспомогательный текст | `Фотограф [Город] — живые эмоции, не постановка` |
| CTA-кнопка | `Обсудить съёмку →` |

**Анимации:**
- H1: кинетическое появление — буквы или слова появляются с fade-in + slide-up при загрузке (задержка 0.3s)
- Подзаголовок: появляется через 0.6s после заголовка
- Кнопка: появляется через 0.9s, лёгкий pulse при idle через 3s

**Технические требования:**
- Фото: WebP, min 2400px по ширине, ≤ 400 КБ после сжатия
- Параллакс на фоне при скролле (фото движется на 20–30% медленнее страницы)
- Мобайл: фото обрезается по центру, текст адаптируется под 375px

---

### [02] ВИДЫ СЪЁМКИ
**Цель:** клиент немедленно узнаёт себя — «это именно то, что мне нужно»

**Макет:** 5 карточек в ряд (десктоп) / 2 колонки (планшет) / 1 колонка (мобайл)

**Карточки:**

| # | Название | Подпись для клиента | Фото |
|---|---|---|---|
| 1 | Портрет | Для тех, кто хочет увидеть себя настоящего — без масок и поз | Портретное фото |
| 2 | Семейная съёмка | Дети вырастают быстро. Снимем момент, который останется навсегда | Семья с детьми |
| 3 | Лав стори | Ваша история в кадрах — без наигранности, только живые эмоции | Пара |
| 4 | Коммерческая | Фото для бизнеса, которое продаёт: команда, офис, бренд | Бизнес-съёмка |
| 5 | Предметная | Продукт выглядит так, что его хочется купить | Предметное фото |

**Структура одной карточки:**
- Фото (aspect-ratio: 3/4, портретная ориентация)
- Категория (uppercase, мелкий шрифт, цвет акцента)
- Название (H3, крупно)
- Описание (1–2 строки, для клиента)
- При hover: лёгкое масштабирование фото (scale 1.04) + появляется кнопка `Смотреть работы →`

**Анимации:**
- Карточки появляются stagger-эффектом при скролле до блока (задержка 0.1s между карточками)

---

### [03] ПОРТФОЛИО
**Цель:** влюбить клиента в стиль и уровень работ

**Макет:** Bento Grid — асимметричная сетка (CSS Grid)
- Десктоп: 3 колонки, некоторые фото занимают 2 ячейки (горизонтальные / вертикальные)
- Мобайл: 2 колонки, равные ячейки

**Содержание:**
- 12–15 лучших работ, mix всех направлений (3 портрета / 3 семьи / 2 лав стори / 2 коммерческих / 2 предметных)
- Единая обработка — стиль должен читаться в сетке как единое целое
- НЕТ водяных знаков

**Заголовок блока:**
```
Избранные работы
```

**Интерактив:**
- Клик по фото — полноэкранный lightbox с навигацией (стрелки / свайп)
- Hover: лёгкое затемнение + иконка «открыть» (loop или лупа)
- View Transitions API для плавного открытия lightbox

**CTA после галереи:**
```
Хочу такие же фото → Написать в Telegram
```

### [04] О ФОТОГРАФЕ
**Цель:** личный контакт, симпатия, «я хочу, чтобы именно этот человек меня снял»

**Макет:** 2 колонки — фото слева (40%), текст справа (60%) / на мобайле стек

**Фото фотографа:**
- Живое, не студийное — в процессе работы или в жизни
- Формат: вертикальный портрет или квадрат
- Без костюма, без пафоса

**Тексты (шаблон — финальный текст написать отдельно):**

```
Заголовок: Привет, я Аркадий

Основной текст (4–6 предложений):
— Почему стал фотографом (личная история, не резюме)
— Что для меня важно в работе с людьми
— Как я работаю: «я не ставлю людей — я наблюдаю и ловлю»
— Одна личная деталь (семья, увлечение)

Факты (3–4 строки с цифрами):
• [X] лет в фотографии
• [X]+ завершённых съёмок  
• Работаю в [Город] и выезжаю
• Отвечаю в течение часа
```

**CTA после блока:**
```
Если вам откликается то, что я делаю — просто напишите
[Написать в Telegram]
```

**Анимация:** текст и фото появляются при скролле с fade-up + небольшим смещением

### [05] КАК ПРОХОДИТ СЪЁМКА
**Цель:** снять страх неизвестности, ответить на «что мне делать, как это будет»

**Макет:** горизонтальная линия шагов (десктоп) / вертикальный список (мобайл)

**Заголовок блока:**
```
Как это работает
```

**4 шага:**

| # | Шаг | Описание |
|---|---|---|
| 01 | Знакомство | Пишете в Telegram — обсуждаем идею, дату, место. Никаких бланков и формальностей. |
| 02 | Подготовка | Я помогу с выбором локации, образа и расскажу как подготовиться. За день пришлю напоминание. |
| 03 | Съёмка | Никакого «замрите». Гуляем, смеёмся, живём момент — я просто рядом с камерой. |
| 04 | Готовые фото | Через [N] дней вы получаете галерею. Скачиваете, сохраняете, вспоминаете. |

**Дизайн шагов:**
- Большой номер (100px+, цвет акцента или тонкий outline)
- Название шага (H3)
- Описание (2–3 строки, тёплый разговорный тон)
- Разделитель между шагами (линия или стрелка)

### [06] ОТЗЫВЫ
**Цель:** социальное доказательство — «другие уже рискнули и не пожалели»

**Заголовок блока:**
```
Что говорят клиенты
```

**Формат карточки отзыва:**
- Фото клиента (круглый аватар, 60px)
- Имя + тип съёмки (Мария, семейная съёмка)
- Текст отзыва — история, не оценка (3–5 предложений)
- Необязательно: скриншот из мессенджера как подтверждение

**Минимум:** 6 отзывов

**Макет:** горизонтальная карусель (десктоп — 3 видно одновременно) / по одному (мобайл)

**Требования к содержанию отзывов:**
- Не «всё понравилось», а конкретная история
- Хотя бы 2–3 отзыва включают упоминание страха до съёмки и результата
- Хотя бы 1 видео-отзыв (если есть)

**CTA после отзывов:**
```
Хочу тоже → Узнать детали
```

### [07] СТОИМОСТЬ
**Цель:** убрать барьер «а сколько это стоит», дать понять что доступно

**Заголовок блока:**
```
Стоимость
```

**Структура:** 3 пакета (якорный эффект — средний выглядит оптимальным)

| | Пакет 1 | Пакет 2 ★ Популярный | Пакет 3 |
|---|---|---|---|
| Название | `Прогулка` | `История` | `День` |
| Продолжительность | 1 час | 2 часа | 4 часа |
| Готовых фото | 20–30 | 50–70 | 100+ |
| Срок сдачи | 14 дней | 14 дней | 21 день |
| Ретушь | Да | Да | Да |
| Галерея онлайн | Да | Да | Да |
| Цена | `от [X] ₽` | `от [X] ₽` | `от [X] ₽` |
| CTA | `Выбрать` | `Выбрать` | `Выбрать` |

**Под таблицей — короткая заметка:**
```
Точная стоимость зависит от локации и деталей.
Для коммерческой и предметной съёмки — индивидуальный расчёт.
Написать и узнать цену →
```

**Важно:**
- Средний пакет визуально выделен (рамка, метка «Популярный»)
- Цены «от X» — не точные, чтобы осталась возможность обсуждения
- Кнопки ведут прямо в Telegram

### [08] FAQ
**Цель:** снять последние возражения до контакта

**Заголовок блока:**
```
Частые вопросы
```

**Формат:** аккордеон (accordion) — вопрос + стрелка, при клике разворачивается ответ

**7 вопросов и ответов:**

| Вопрос | Ответ (тон — разговорный, тёплый) |
|---|---|
| Сколько готовых фотографий я получу? | Зависит от пакета — от 20 до 100+ снимков. Все прошедшие отбор фото будут обработаны. Не количество, а качество. |
| Как долго ждать готовые снимки? | Обычно 10–14 дней. Если нужно срочно — обсудим отдельно. |
| Нужно ли готовиться? Что надеть? | Да, я помогу. После брони пришлю памятку — что взять, как одеться, чего избегать. |
| Что если дети не хотят фотографироваться? | Это нормально. Я не заставляю позировать — просто наблюдаю. Лучшие кадры всегда получаются когда дети забывают про камеру. |
| Как происходит оплата? | Предоплата 30–50% для брони даты. Остаток — в день съёмки или после. Карта, СБП, наличные. |
| Вы работаете только в [Город]? | Базируюсь в [Город], но легко выезжаю. Выезд за пределы города — по договорённости. |
| Можно ли перенести дату? | Да, если предупредить минимум за 3 дня. Форс-мажоры — обсудим индивидуально. |

**Ключевые слова:** тёплый · воздушный · живой · настоящий · премиум без пафоса  
**Эстетика:** тёплый минимализм — много белого пространства, крупная типографика, фото говорят сами за себя

## 6. АНИМАЦИИ И ИНТЕРАКТИВ

### Обязательные анимации

| Элемент | Анимация | Описание |
|---|---|---|
| Hero H1 | Кинетическое появление | Слова появляются последовательно: fade-in + translateY(20px) → 0, delay 0.15s между словами |
| Hero подзаголовок | Fade-in | 0.6s после H1 |
| Hero кнопка | Fade-in + pulse idle | Появляется через 0.9s, тихий pulse через 3s бездействия |
| Секции при скролле | Reveal-анимация | fade-in + translateY(30px) → 0, duration 0.6s, easing ease-out |
| Карточки услуг | Stagger reveal | Появляются с задержкой 0.1s между карточками при скролле |
| Фото в галерее | Hover scale | transform: scale(1.04), transition 0.4s |
| Кнопки | Hover инверт | background меняется на тёмный/светлый, transition 0.25s |
| Шапка при скролле | Blur появляется | backdrop-filter: blur(16px) + полупрозрачный фон |
| Lightbox | View Transition | Плавное открытие фото из карточки галереи |

### Желательные анимации

| Элемент | Анимация | Приоритет |
|---|---|---|
| Hero фон | Parallax скролл (20–30% от скорости страницы) | Высокий |
| Кастомный курсор | Маленький круг, на фото — увеличивается с текстом «View» | Средний |
| Цифры в блоке «Обо мне» | Count-up анимация при появлении | Средний |
| Grain texture overlay | SVG шум поверх hero и фоновых блоков (opacity: 0.04) | Средний |
| Кнопка Telegram (mobile) | Плавающая sticky-кнопка внизу экрана | Высокий |

### НЕ делать
- Музыка с автовоспроизведением
- Анимации дольше 0.8s
- Параллакс на мобильных (отключить)
- Блокирующие прелоадеры дольше 1s

**Стиль кнопок:**
- Размер: padding 14px 32px
- Скругление: 4–6px (не таблетки, умеренное)
- Шрифт: Inter, 500, 14–15px, uppercase с letter-spacing
- Hover: инверсия цвета за 0.25s

**Ссылка на Telegram:** `https://t.me/[username]` — открывать в новой вкладке

### Форма (если нужна — опционально)

- Поля с минимальным визуальным весом (нет тяжёлых рамок)

## 9. ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ

### Платформа (рекомендация)
**Tilda** (план Business, ~1 250 ₽/мес)
- Zero Block — для кастомных секций (Hero, Виды съёмки)
- Стандартные блоки — для галереи, FAQ, отзывов
- Встроенная адаптивность и SEO

**Альтернатива для максимального дизайна:** Framer

### Производительность
| Метрика | Цель |
|---|---|
| PageSpeed Score (мобайл) | ≥ 80 |
| PageSpeed Score (десктоп) | ≥ 90 |
| LCP (Largest Contentful Paint) | ≤ 2.5 сек |
| CLS (Cumulative Layout Shift) | ≤ 0.1 |
| Все изображения | WebP, lazy load |
| Hero-изображение | Preload (`<link rel="preload">`) |

### SEO
- `<title>`: `Фотограф [Имя] — Портрет, Семья, Лав стори | [Город]`
- `<meta description>`: 155 символов с ключевыми словами и городом
- Alt-теги для всех фото (описательные, с геолокацией)
- Структурированные данные: `schema.org/LocalBusiness` + `schema.org/Photographer`
- Подключить: Яндекс.Метрика + Google Analytics 4 + Google Search Console + Яндекс.Вебмастер

### Мобильная версия
- Mobile-first подход
- Минимальный размер шрифта: 14px
- Touch-таргеты (кнопки, ссылки): минимум 44×44px
- Свайп в галерее и карусели отзывов
- Sticky Telegram-кнопка внизу экрана
- Параллакс-эффекты — отключить на мобайле

### Безопасность и базовые требования
- SSL-сертификат (HTTPS): обязателен
- Защита изображений: `pointer-events: none` + `user-select: none` на фото (базовая защита)
- Cookie-уведомление: простой баннер при первом визите
